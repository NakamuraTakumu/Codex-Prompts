# 論文レビュー基準

## 適用

- **用途**: レビュー対象に応じて必要な評価面だけを選ぶ。
- **出力単位**: 所見は選択した **評価面** ごとに出力し、各評価面内では `major`、`moderate`、`minor`、`nit` の順に並べる。
- **省略規則**: ユーザー指示で考慮する **評価面** が制限される場合、対象外の評価面は出力しない。
- **構造化文書レビュー**: LaTeX source、Markdown note、論文差分の構造、冗長性、用語・表記、情報保持を確認するときは、この基準を **基準入力** として `../../common/structured_document_rule.md` の **レビュー検査** に渡す。
- **非閉鎖性**: この基準は網羅 checklist ではない。分類外の重要 issue も影響に基づいて報告する。

## 評価原則

- **優先順位**: 表現の polish より、正確性、論理的一貫性、根拠、再現性、先行研究との位置づけ、投稿規定を優先する。
- **判断軸**: 読者理解、正確性、投稿品質への影響で severity を決める。
- **構造化文書 issue**: `../../common/structured_document_rule.md` に基づく issue も、文書構造そのものではなく論文理解、正確性、投稿品質への影響で severity を決める。
- **非判断軸**: 修正案の有無や修正の容易さだけでは severity を下げない。

## 評価面

この section では、各評価面と、**定理系環境** から参照する子スコープ **定理系環境の役割** を定める。

- **主張と貢献**:
  - **対象**: 主要貢献、結論、貢献の位置づけ。
  - **見る問題**: 貢献不明、過大主張、根拠から導けない結論。
- **構成と論理**:
  - **対象**: 節、段落、定義、仮定、記法、用語。
  - **見る問題**: 論理の飛躍、導入不足、段落冒頭の主張を支えない説明。
- **定理系環境**:
  - **対象**: definition、theorem、proposition、lemma、corollary、proof、remark、example。
  - **見る問題**: **定理系環境の役割** との不一致、statement と proof の不対応。
- **方法、根拠、妥当性**:
  - **対象**: 方法説明、実験、分析、例、制限、前提条件、因果表現、一般性表現。
  - **見る問題**: 手順の追跡不能、根拠不足、制限の未提示、根拠のない因果関係や一般化。
- **先行研究と引用**:
  - **対象**: 先行研究、参考文献の該当箇所、差分説明、引用文、引用と主張の接続。
  - **見る問題**: 重要文献の欠落、名前の付け替えに留まる差分説明、引用または参考にした主張と参考文献の内容の不一致、引用が主張を支えていない箇所、引用文の不自然な組み込み。
- **BibTeX と bibliography**:
  - **対象**: `.bib` file、citation key、entry type、author、title、year、venue、DOI、URL、bibliography output。
  - **見る問題**: citation key の不一致、DOI または URL の欠落、必要 field の欠落、著者名・表題・年・出版先の誤り、重複 entry、不適切な entry type、本文 citation と bibliography output の不対応。
- **表現と言語**:
  - **対象**: 文、参照表現、用語、略語、記号、文体、日本語または英語の学術表現。
  - **見る問題**: 指示語や修飾関係の曖昧さ、用語不統一、対象言語として目立つ不自然さ、根拠に合わない主張の強さ。
- **冗長性**:
  - **対象**: LaTeX source、Markdown note、論文差分の重複する主張、説明、例、規則、見出し、note、legacy 記述。
  - **見る問題**: `../../common/structured_document_rule.md` の **冗長性の排除** に反する問題。
- **構造性**:
  - **対象**: LaTeX source、Markdown note、論文差分の section、段落、list、定理系環境、command、macro、参照導線。
  - **見る問題**: `../../common/structured_document_rule.md` の **構造化** に反する問題。
- **厳密性**:
  - **対象**: LaTeX source、Markdown note、論文差分の定義、仮定、条件、量化、記法、用語、判断基準、出力契約。
  - **見る問題**: `../../common/structured_document_rule.md` の **厳密化** に反する問題。
- **図、式、LaTeX、投稿規定**:
  - **対象**: 図、式、caption、label、参照、rendered PDF、抽出 PDF テキスト、style file、class file、bibliography style、front matter、分量。
  - **見る問題**: 本文との未接続、番号や参照の破損、source では見えない表示上の曖昧さ、command leakage、投稿規定違反、手動整形と style file の衝突。

### 定理系環境の役割

この section では、各定理系環境の役割と混入禁止内容を定める。

- **statement 共通**: theorem、proposition、lemma、corollary は、仮定、量化、結論、適用範囲を statement 内または直前文脈から回復できる形で述べる。
- **definition**:
  - **役割**: 用語、記号、対象、構成、条件を導入し、以後の使用範囲を固定する。
  - **混入禁止**: 証明を要する性質、主要主張、例外的事実、未証明の帰結。
- **theorem**:
  - **役割**: 論文の主要または独立した数学的主張を述べる。
  - **混入禁止**: 証明方針、直観、例、補足説明。
- **proposition**:
  - **役割**: theorem より局所的または補助的だが、独立して証明される主張を述べる。
  - **混入禁止**: proof 内の中間計算、definition の一部、remark 的な解釈。
- **lemma**:
  - **役割**: 後続の theorem、proposition、proof を支える補助主張を述べる。
  - **混入禁止**: 論文の主要結論、後続で使われない孤立主張。
- **corollary**:
  - **役割**: 直前または明示参照した theorem、proposition、lemma から短く従う帰結を述べる。
  - **混入禁止**: 新しい仮定、独立した証明を要する主張、主要結果の再定義。
- **proof**:
  - **役割**: 対応する statement の全結論を、仮定と既出結果から正当化する。
  - **混入禁止**: statement にない新しい主張、未導入記法、証明と無関係な説明。
- **remark**:
  - **役割**: 主張の理解を助ける補足、直観、関係、制限、使い方を述べる。
  - **混入禁止**: 後続論理に必須の定義、仮定、定理、証明の一部。
- **example**:
  - **役割**: 定義、仮定、結果の具体例、適用例、反例を示す。
  - **混入禁止**: 一般主張、証明の代替、後続で必須になる未定義の構成。

## 重大度

- **判定単位**: issue が単独で最終稿に残った場合の影響。
- **非判定軸**: 他の issue との相対的な目立ちやすさ、修正の容易さ、修正案の有無。
- **判定順序**: 複数の重大度条件を満たす場合は最大影響で判定する。
- **`major`**: 論文の正確性、主要主張、証明、再現性、先行研究理解、投稿規定遵守、参考文献の同定可能性を壊す。
- **`moderate`**: `major` には至らないが、読者または査読者の理解、検証、引用確認、投稿品質を実質的に損なう。DOI または URL の欠落、重要な bibliography field の欠落はここに含める。
- **`minor`**: 局所的な明瞭性、用語統一、限定的な曖昧性、軽い bibliography metadata の改善に関わるが、主張、検証、投稿可否への影響は小さい。
- **`nit`**: 意味、検証、投稿品質に実質的な影響がない。TeX source のメンテナンス性だけに関わり、rendered output、参照、build、投稿規定に影響しない問題もここに含める。
