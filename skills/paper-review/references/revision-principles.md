# 論文修正原則

原稿修正時の安全制約として使う。

## 適用

- **修正対象**: 何を修正対象にするかは、[criteria.md](criteria.md) の評価面と重大度に従う。
- **構造化文書 revision**: section、段落、list、定理系環境、command、macro、Markdown note の構造、冗長性、用語・表記、情報保持を修正する場合は、この原則と [criteria.md](criteria.md) を **基準入力** として `../../common/structured_document_rule.md` の **リバイズ** に渡す。
- **構造修正の上限**: `../../common/structured_document_rule.md` の **境界修正**、**構造修正**、**到達可能性** は、下記の **範囲保持** に反しない場合だけ適用する。指定範囲、環境種別、節構成、段落構成を変える修正は、明示依頼がない限り所見または安全な修正案に留める。
- **非対象**: 大幅な構成変更、論文の主張を意図的に変える修正。

## 修正制約

- **意味保持**: 主張、含意、論理関係、帰属、確実性、限定条件を暗黙に変えない。
- **最小修正**: 広い書き換えより、安全な局所修正を優先する。収まらない問題は広く書き換えない。
- **内容追加禁止**: 求められていない主張、根拠、比較、例、留保、解釈を追加しない。
- **範囲保持**: 安全な局所修正に必要でない限り、指定範囲、環境種別、節構成、段落構成を変えない。
- **一貫性保持**: 原稿の既存文体、用語、記法、著者の声、近傍の反復 pattern に合わせる。
- **既存 TODO 保持**: 明示的に依頼されていない TODO marker は、内容を埋めず、削除せず、解決済みとして扱わない。
- **引用文と macro**: command の差し替えだけで済ませず、文全体と修辞上の接続を確認する。

## TODO marker

- **適用条件**: `revision` mode で、最小修正に収まらない問題、意味保持と衝突する問題、著者判断が必要な問題がある。
- **扱い**: TODO marker は未解決 issue の表示として原稿へ追加する。新しい主張、根拠、例、比較、解釈、結論は追加しない。
- **形式**: 追加 package なしの LaTeX で使える `\marginpar{\footnotesize TODO: ...}` を使う。`...` は未解決の作業を命令形または名詞句で短く書き、理由や長い説明を入れない。
- **配置**: 対応箇所の近傍に置く。`\marginpar` が使えない文脈では近傍の本文段落に置き、近傍にも置けない場合は原稿に追加せず Markdown note の未解決 issue として残す。
- **記録**: 追加した TODO marker は Markdown note の **適用済み修正** または **未解決 issue** に記録する。

## 表示結果

- **確認条件**: rendering 結果が修正対象またはリスク対象である場合。
- **確認規則**: 確認条件を満たす場合は source だけで判断せず、表示結果を確認する。
- **対象例**: figure、caption、front matter、page break、visual annotation、layout-sensitive construct。
