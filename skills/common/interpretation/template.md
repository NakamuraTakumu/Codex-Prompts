# [Context Name] Interpretation

参照元: `../structured_artifact_rule.md`

この file は、`structured_artifact_rule.md` の `[*]` 付き抽象語を、特定文脈へ割り当てるための interpretation file template である。rule 本体ではない。

## 用途

* **目的**: [文脈名] task で、抽象語の解釈観点を固定し、task 中の推論の揺れを減らす。

* **対象**: [文脈名] のうち、task の **根スコープ** に含めるもの。

## 記述形式

各解釈 section は、必要な field だけを使う。

* **割当**:
  * 抽象語に対応させる文脈内の対象。
  * 名詞句で列挙する。
* **補足**:
  * 抽象 rule から推論できるが、重要または推論しづらいため明示すること。
* **追加**:
  * 抽象 rule からは推論できないが、抽象 rule の推論と矛盾しない文脈固有のこと。
* **例外**:
  * 抽象 rule からの自然な推論を上書きすること。
  * なければ `なし。` と書く。

補足、追加、例外には、次を書かない。

* **補足に書かないこと**: 抽象 rule から容易に推論でき、文脈上の誤解も起きにくいこと。
* **追加に書かないこと**: 抽象 rule の推論と矛盾すること、または単なる割当候補の再掲。
* **例外に書かないこと**: 抽象 rule と矛盾しない文脈固有の追加情報、または通常の注意事項。
