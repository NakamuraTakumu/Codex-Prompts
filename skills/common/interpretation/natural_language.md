# Natural Language Interpretation

参照元: `../structured_artifact_rule.md`

この file は、`structured_artifact_rule.md` の `[*]` 付き抽象語を、自然言語で意味のまとまりを作る文脈へ割り当てるための interpretation file である。rule 本体ではない。

記法や file 形式に固有の解釈は扱わない。

## 用途

* **目的**: 自然言語で構成される task で、文や段落などのまとまりを使って、抽象語の解釈観点を固定する。

* **対象**: 文、句、段落、項目、見出し付きのまとまりなど、自然言語で働きとつながりを表す範囲のうち、task の根になるもの。

## 解釈スコープ

### 成果物

* **割当**: 文章の範囲、文章全体、対象範囲、修正対象、参照する前後文脈。
* **補足**: 読み取り対象、変更対象、参照対象は分けて扱う。前後文脈が解釈、例外、補足、結論を固定する場合は、変更対象でなくても参照対象に含める。
* **追加**: なし。
* **例外**: なし。

### スコープ

* **割当**: 意味のまとまり、文章全体、章、段落、項目、文、句。
* **補足**: 文章のまとまりは、意味の切れ目として扱う。文、句、段落の違いによって、担える働きと示せるつながりは変わる。
* **追加**: なし。
* **例外**: 記法や file 形式だけで作られる境界は、この file では割当に使わない。

### 名前

* **割当**: 文書内の対象、まとまり、概念、役割を参照する表現、見出し、用語、ラベル、指示表現、略称、言い換え表現。
* **補足**: 対象の文章、参照する前後文脈、依頼文、外部参照の範囲から、task で見る範囲を固定する。指示表現や言い換え表現を使う場合は、参照先を前後のまとまりから回復できるようにする。
* **追加**: なし。
* **例外**: 外部基準で決まっている用語や名称は変更せず、必要なら周辺説明で役割を補う。

### 責務

* **割当**: 読者に対する働き、冒頭の要点、導入、まとめ、つなぎ、参照表現。
* **補足**: 冒頭の要点はまとまりの働きを示し、導入は読む順序、比較軸、列挙対象を示す。まとめは下位のまとまりの共通点、結論、未解決事項をまとめる。つなぎと参照表現は文章上のつながりを示す。
* **追加**: なし。
* **例外**: 前提、主張、理由、結論、例外、補足は、独立した働きとして機械的に列挙しない。1 つのまとまりの働きを支える関係なら、自然言語で関係を示す。

### 参照関係

* **割当**: まとまり、表現、働きの間の前後関係、包含、依存、対比、補足、例外、帰結、参照先。
* **補足**: 見出しや項目だけで回復しにくい関係は、自然言語で示す。親子関係、兄弟関係、前後依存、参照先、例外、本則との差分は、必要な範囲で明示する。
* **追加**: なし。
* **例外**: 接続表現の有無だけではつながりを判断しない。文章上のつながりが、実際の意味のまとまりと矛盾している場合は、意味のまとまりを優先して見直す。
